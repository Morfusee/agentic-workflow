NGN-645 · CSP frame-ancestors

Finance iframe now embeds from HTTPS local dev. Port-wildcard pattern (https://*.dev1.ngnair.com:*) deployed to staging. QA: FAIL then PASS.

This was the most iterative area this week. HTTPS local dev tickets were created for location-frontend and location-admin to match the marketplace-frontend mkcert pattern. Josh implemented both, adding justfiles and updating package.json dev scripts with --experimental-https. The CSP frame-ancestors fix for Finance iframe embedding went through two QA rounds. The initial approach was to switch location-frontend to HTTPS so the existing CSP wildcard would match. QA testing found the wildcard failed on non-default ports because the CSP spec does not match ports on hostname wildcards. Josh applied a port-wildcard pattern with colon-star in the CSP, updated the CapRover env var, and re-testing on both develop and staging returned PASS. All three tickets are Done. An interesting side effect: NGN-650 and NGN-651 also closed the original NGN-645 indirectly. The finance-frontend CSP header frame-ancestors https://*.dev1.ngnair.com already matches any HTTPS origin under that wildcard. Once a developer runs location-frontend over HTTPS locally, embedded finance iframes load without CSP violations. The port-wildcard fix in NGN-645 was the final piece that made the full flow work end-to-end.
