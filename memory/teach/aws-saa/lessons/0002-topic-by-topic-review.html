1. IAM & AWS CLI

The short version:

  • Users = long-term credentials (password + access keys). One per person or service.
  • Groups = collections of users. Attach policies here, not to individual users.
  • Roles = temporary credentials (via STS). Used for EC2 instances, cross-account, federated access.
  • Policies = JSON: Effect (Allow/Deny) + Action + Resource + optional Condition.
  • Key rule: Explicit Deny always wins over Allow.
  • Permissions Boundary = max ceiling within an account. SCP = max ceiling across all accounts in an Organization.

2. EC2 Fundamentals

The short version:

  • Instance type naming: [family][gen].[size] — m=general purpose, c=compute, r=memory, i=storage, g=GPU
  • Security Groups: instance-level, stateful, allow rules only. NACLs: subnet-level, stateless, allow+deny, ordered.
  • Purchase options: On-Demand (flexible), Reserved (steady, 1/3yr, up to 72% off), Spot (fault-tolerant, 90% off, can be interrupted), Dedicated Host (compliance, sockets/cores visibility)
  • Placement Groups: Cluster (low latency, same AZ), Spread (fault isolation, 7/AZ), Partition (distributed apps, rack-level isolation)
  • Stop vs Terminate: Stop preserves EBS, changes public IP. Terminate destroys instance (unless disableApiTermination is set)

3. ELB & Auto Scaling Groups

The short version:

  • ALB (L7): HTTP/HTTPS, path/host-based routing, WAF, sticky sessions, container support (ECS).
  • NLB (L4): TCP/UDP/TLS, static IP per AZ, extreme performance, preserves client IP.
  • GLB (L3/4): Transparent gateway + LB for third-party appliances (GENEVE protocol).
  • Cross-zone LB: ALB = free & on by default. NLB = charged & off by default.
  • Sticky sessions: Cookie-based routing to the same target. Cookie expires → different target.
  • Connection draining (deregistration delay): lets in-flight requests complete before removing target.
  • ASG: Min/Desired/Max. Target tracking is easiest scaling policy. Launch Templates > Launch Configs.
Your application MUST receive traffic with the original client IP preserved. Which load balancer type can you use?
  • Application Load Balancer only
  • Network Load Balancer only
  • Both ALB and NLB preserve the client IP
  • Neither — client IP is always replaced

  • The ALB has session replication, so the cart follows the user
