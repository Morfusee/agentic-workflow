Lesson 1: IP Addresses & Subnetting

How your server and VMs talk to the internet — and each other.

Why this matters for your KimSufi

When you open the OVH control panel after ordering your KimSufi server, you'll see something like:

IPv4: 51.195.46.19
IPv6: 2001:41d0:0401:01d2::/64

That's your server's public identity on the internet. Your Proxmox VMs will also need public identities if you want them to serve web apps, handle email, or be reachable from outside. Understanding IP addressing is the foundation of every network decision you'll make.

What is an IP Address?

Think of an IP address like a mailing address for a device. For a letter to reach you, it needs your street address. For data to reach a server, it needs its IP address.

IPv4 addresses (the ones we'll focus on) are 32-bit numbers, written as four decimal numbers separated by dots — each between 0 and 255:

51 . 195 . 46 . 19

That's your KimSufi's main IP. Every device on the internet needs a unique public IPv4.

Two kinds of IPs
Public IP — routable on the internet. Your server's main IP is public.
Private IP — only usable inside a local network. Ranges: 10.x.x.x, 172.16.x.x, 192.168.x.x. Your home devices (laptop, phone) use private IPs behind your router.
At OVH, your VMs will use private IPs on the Proxmox bridge (e.g. 192.168.0.2) with a public Additional IP mapped on top for internet access.

Subnetting & CIDR — the part that matters most

An IP address alone isn't enough. You also need to know which part of the address identifies the network and which part identifies the specific device. That's where the subnet mask comes in.

Here's the notation you'll see constantly in OVH guides:

51.195.46.19/32    <- your server's main IP
46.105.135.96/28   <- a block of 16 IPs
192.168.0.1/24     <- typical bridge network
100.64.0.1         <- OVH's gateway (no /xx needed, it's a single IP)

The /xx is CIDR notation — how many bits are reserved for the network portion. The remaining bits identify hosts on that network.

CIDRSubnet MaskTotal IPsUsableContext
/32255.255.255.25511Your server's main IP
/30255.255.255.25242Point-to-point link
/28255.255.255.2401613OVH Additional IP block
/24255.255.255.0256254Home network / Proxmox bridge
Why /32 for your server?
OVH assigns the server IP as a /32 — it's a single host address. The gateway is a separate IP (100.64.0.1) on a special "carrier grade" network. This is unusual but important: it means your server doesn't share a subnet with anything else. Your VM bridge uses a private /24 (192.168.0.1/24) internally.

The Special Trio: Network, Gateway, Broadcast

Every subnet reserves three addresses that cannot be assigned to a device:

46.105.135.96    <- Network address  (first, all host bits = 0)
46.105.135.97    <- First usable IP
...
46.105.135.109   <- Last usable IP
46.105.135.110   <- Gateway          (penultimate IP — OVH convention!)
46.105.135.111   <- Broadcast address (last, all host bits = 1)
OVH's twist on gateway placement
Most providers put the gateway as the first usable IP. OVH puts it as the penultimate (second-to-last) IP in a block. If you buy a /28 block from OVH and use it in a vRack, the gateway is .110 (if the network is .96). Always check their docs.

How this maps to your Proxmox config

Here's the networking picture you're building toward:

        Internet
           |
   [OVH Gateway: 100.64.0.1]
           |
   [KimSufi Server - main IP: 51.195.46.19/32]
        |
   [vmbr0 bridge - 192.168.0.1/24]
     /    |     \
   VM1   VM2    VM3
  .0.2   .0.3   .0.4
  +       +       +
  |       |       |
AddIP1  AddIP2  AddIP3

Each VM gets a private IP on the bridge (192.168.0.x) plus an Additional (public) IP routed to it. The VM uses the private IP for default traffic and the Additional IP as the source address for outgoing packets.

Interactive Quiz

Check your understanding

  1. Your OVH dashboard says your server IP is 51.195.46.19/32. What does the /32 mean?
    2. It means this is a single host address. All 32 bits identify the device — there are zero host bits, so no other device shares this subnet.
  2. You buy an Additional IP block 46.105.135.96/28 from OVH. How many usable IPs do you get for your VMs?
    3. 13 usable IPs. /28 = 16 total addresses. OVH reserves: network (.96), gateway (.110), and broadcast (.111). That's 3 reserved, leaving 13 assignable (.97–.109).
  3. A VM on your Proxmox bridge has IP 192.168.0.5. Can the internet directly reach it at that address? Why or why not?
    4. No. 192.168.x.x is a private range — not routable on the public internet. The VM needs an Additional public IP mapped to it, or NAT through the host, to be internet-reachable.
  4. In OVH's vRack, why is the gateway at the penultimate IP (.110 in a /28) rather than the first usable IP?
    5. That's OVH's convention — the penultimate address is always the gateway, the last is the broadcast. This differs from most providers (who use the first usable as gateway). Always verify with OVH's documentation for your block.
