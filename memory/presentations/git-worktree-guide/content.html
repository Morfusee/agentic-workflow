Git Worktree Session Notes

A clean, practical guide for keeping the main repository stable while doing AI-agent work in a separate Git worktree and branch.

Date 2026-05-21
Type Research
Tags git, worktree, dev-workflow, ai-agents
TL;DR

For AI-agent work, keep your original repo on main, create a separate worktree from origin/main, give it a dedicated branch like agent/workspace, and push the new branch with git push -u origin HEAD the first time.

At a glance
Preferred base
origin/main
Suggested branch
agent/workspace
Safe first push
git push -u origin HEAD
Mental model
Branch = timeline, worktree = folder

Goal

Create a separate Git worktree for AI-agent work so the main repo stays clean while the agent works in an isolated folder and branch.

Key Concepts

A Git worktree is a separate working folder connected to the same Git repository.

Each worktree can only have one branch checked out at a time, but it still has access to the repository's local and remote branches.

$HOME/Documents/Programming/website
  -> main workspace, usually on main

$HOME/Documents/Programming/website-agent
  -> AI-agent workspace, usually on an agent/task branch

LazyGit Worktree Options

When LazyGit shows the worktree creation menu, use the normal branch-based option for coding work.

Use this
Create worktree from ref

This is the right choice for normal development and commits.

Avoid unless intentional
Create worktree from ref (detached)

Use detached mode only when inspecting or testing a specific commit without normal branch-based work.

Recommended AI Worktree Setup

Create the worktree from main or, more commonly, origin/main.

Suggested defaults:

  • Base ref: origin/main
  • Worktree path: $HOME/Documents/Programming/website-agent
  • Branch name: agent/workspace
  • Task-specific variant: agent/llms-txt-cms
git worktree add ../website-agent -b agent/workspace origin/main

Fetch All Branches

Before creating worktrees or choosing remote branches, refresh your local branch view first.

git fetch --all --prune
git branch -a

Important Worktree Rule

The same local branch cannot normally be checked out in multiple worktrees at the same time.

website/        -> main
website-agent/  -> main

That setup is usually blocked. Instead, create a fresh branch from main for the new worktree:

git worktree add ../website-agent -b agent/workspace main

Switching Branches Inside a Worktree

A worktree can switch to another branch, but only if that target branch is not already active in a different worktree.

git switch feat/some-branch

If the branch is already checked out elsewhere, Git will reject the switch.

Push Error Encountered

This error came from the local branch tracking the wrong upstream branch:

fatal: The upstream branch of your current branch does not match the name of your current branch.

Typical mismatch:

local:  feat/CU-86d32pnat-cms-managed-content-served-at-llmstxt
remote: origin/main

The local feature branch was pointing at origin/main instead of a same-named remote branch.

Fix Current Branch

Push the current branch to a same-named branch on origin and set upstream in one step:

git push -u origin HEAD

After that, normal pushes work again:

git push

Permanent Git Config Recommendation

If you frequently create feature or worktree branches, set Git to behave more helpfully by default:

git config --global push.default current
git config --global push.autoSetupRemote true

Verify the settings:

git config --global push.default
git config --global push.autoSetupRemote

Expected output:

current
true

Avoid This Unless Intentional

Do not use the command below unless you explicitly want to push your current branch directly into remote main:

git push origin HEAD:main

That means:

Push my current branch's commits into origin/main.

For AI-agent branches or feature branches, that is usually the wrong target and can be risky.

Practical Workflow

  1. Stay on main in the original repository.
  2. Fetch all branches with git fetch --all --prune.
  3. Create a new worktree from origin/main.
  4. Give the worktree its own branch, such as agent/workspace.
  5. Open that worktree folder in the AI agent or editor.
  6. Do the work there.
  7. Push with git push, or use git push -u origin HEAD the first time.

Recommended Mental Model

Branch = timeline of commits
Worktree = physical folder showing one branch

For AI-agent work, prefer a dedicated worktree and a dedicated task branch:

git worktree add ../repo-agent-task -b agent/task-name origin/main
